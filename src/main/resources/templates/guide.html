Руководство по комментированию

Для того чтобы генератор документации смог правильно извлечь информацию из вашего кода, используйте специальные **документирующие комментарии**, схожие с Javadoc в Java.

Общий формат документирующего комментария

Документирующий комментарий начинается с /** и заканчивается */. Каждая строка между ними, как правило, начинается с *. Комментарий должен располагаться непосредственно перед элементом кода (классом, методом, полем), который вы хотите документировать.


/**
 * Это пример документирующего комментария.
 * Первая строка после /** (или первая после * ) является
 * кратким описанием элемента.
 *
 * Далее может идти более детальное описание,
 * которое может содержать несколько параграфов.
 * Пустая строка внутри комментария создает новый параграф в документации.
 *
 * Здесь начинаются блоки тегов @.
 * @param имя_параметра Описание параметра.
 * @return Описание возвращаемого значения.
 * @throws ИмяИсключения Описание исключения.
 */
public class MyClass {
    // ...
}

Основные части документирующего комментария:

  1. **Краткое описание:** Первая строка (или первое предложение, заканчивающееся точкой). Оно используется в сводках.
  2. **Детальное описание:** Остальная часть текста до первого блока тегов (@...).
  3. **Блоки тегов:** Начинаются со знака @ в начале строки. Каждый тег имеет определенное назначение.

Основные теги, которые распознает генератор:

@param имя_параметра Описание
Описывает параметр метода или конструктора. Должен быть один тег @param для каждого параметра. Имя параметра должно совпадать с именем параметра в коде.
@return Описание
Описывает возвращаемое значение метода. Используется только для методов, возвращающих не void.
@throws ИмяИсключения Описание
Описывает исключение, которое может быть выброшено методом.
@author Имя Автора
Указывает автора класса или файла.
@version Версия
Указывает версию класса или файла.
@see Ссылка
Добавляет ссылку на связанный элемент (другой класс, метод, URL).

Применение комментариев к элементам кода:

  • **Класс/Интерфейс/Enum:** Комментарий ставится непосредственно перед объявлением class, interface, enum или @interface.
  • **Метод:** Комментарий ставится непосредственно перед объявлением метода. Включает теги @param и @return (если применимо).
  • **Поле:** Комментарий ставится непосредственно перед объявлением поля (переменной).

Следуя этим правилам, вы поможете генератору создать полную и полезную документацию для вашего проекта!
